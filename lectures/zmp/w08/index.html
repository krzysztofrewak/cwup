Dokumentacja projektowa
Co możemy znaleźć w dokumentacji?
  • instrukcja uruchomienia projektu lokalnie (setup, konfiguracja, wymagania)
  • opis architektury systemu (diagramy, podział na moduły, zależności)
  • przewodnik po repozytorium (struktura katalogów, kluczowe pliki, konwencje nazewnictwa)
  • opis API i integracji zewnętrznych (endpointy, formaty danych, autoryzacja)
  • opis procesów developerskich (flow pracy z GIT, code review, zasady merge’owania)
  • opis środowisk i konfiguracji (dev, test, staging, produkcja, zmienne środowiskowe)
  • lista najczęstszych problemów i FAQ (znane błędy, obejścia, dobre praktyki)
